Welcome to the **Machines** section. This space serves as a central repository for technical write-ups and walkthroughs of various vulnerable environments I have compromised across different training platforms.

My focus is on documenting the exploitation process, emphasizing the analytical reasoning behind each step. Whether it's a legacy Windows system or a modern Linux cloud environment, these write-ups aim to dissect the vulnerabilities and the configuration flaws that make them exploitable.

### Platforms

The laboratories are organized by platform and difficulty:

* **[Hack The Box (HTB)](./HTB/index.md):** Detailed solutions for HTB machines, ranging from foundational concepts to **Insane** level attack chains.
* **Other Platforms:** *Coming soon.* Future write-ups will include challenges from diverse environments and providers.

### Methodology

Each write-up follows a rigorous offensive security workflow:
1.  **Reconnaissance:** Initial discovery and service enumeration.
2.  **Foothold:** Exploiting the first entry point into the system.
3.  **Privilege Escalation:** Elevating access to Administrative or Root levels.
4.  **Post-Exploitation:** Domain dominance, persistence, or data exfiltration analysis.
